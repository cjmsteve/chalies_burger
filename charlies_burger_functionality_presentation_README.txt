# 찰리스버거 기능 중심 프레젠테이션 제작 안내

## 개요
이 폴더에는 찰리스버거 웹사이트의 기능적 측면에 초점을 맞춘 프레젠테이션 제작을 위한 다음 세 가지 문서가 포함되어 있습니다:

1. **charlies_burger_functionality_presentation.txt**: 프레젠테이션의 기본 구조와 각 슬라이드에 포함될 내용을 개략적으로 설명
2. **charlies_burger_functionality_presentation_guide.txt**: 각 슬라이드의 발표 내용, 시각 자료, 기술적 세부사항에 대한 상세 가이드
3. **charlies_burger_functionality_slide_mockups.txt**: 주요 슬라이드의 레이아웃을 시각적으로 표현한 텍스트 기반 목업

이 문서들은 찰리스버거 웹사이트의 기능적 측면을 효과적으로 소개하는 프레젠테이션을 만들기 위한 기초 자료로 활용할 수 있습니다.

## 사용 방법

### 1단계: 프레젠테이션 개요 검토
`charlies_burger_functionality_presentation.txt` 파일을 먼저 검토하여 프레젠테이션의 전체 구조와 포함될 내용을 파악합니다. 이 문서는 다음 정보를 제공합니다:
- 총 슬라이드 수 (12장)
- 각 슬라이드의 주제
- 각 슬라이드에 포함될 핵심 내용
- 코드 스니펫 예시
- 필요한 시각 자료 제안

### 2단계: 상세 가이드 검토
`charlies_burger_functionality_presentation_guide.txt` 파일을 검토하여 각 슬라이드의 발표 내용과 시각 자료에 대한 상세 지침을 확인합니다. 이 문서는 다음 정보를 제공합니다:
- 각 슬라이드별 발표 내용 스크립트
- 시각 자료 제안
- 기술적 세부사항 설명
- 대상 청중별 조정 사항
- 발표 팁

### 3단계: 슬라이드 목업 참고
`charlies_burger_functionality_slide_mockups.txt` 파일을 참고하여 주요 슬라이드의 레이아웃을 시각적으로 이해합니다. 이 문서는 다음을 제공합니다:
- 6개 주요 슬라이드의 텍스트 기반 레이아웃 목업
- 요소 배치에 대한 시각적 가이드
- 디자인 요소 참고사항 (색상 코드, 폰트 크기, 이미지 처리 등)

### 4단계: 프레젠테이션 제작
위 세 문서를 참고하여 PowerPoint 또는 다른 프레젠테이션 소프트웨어를 사용해 실제 프레젠테이션을 제작합니다:

1. 새 프레젠테이션 파일 생성
2. 브랜드 색상(#FF6B00, #333333)을 적용한 테마 설정
3. 개요에 따라 12개 슬라이드 생성
4. 가이드와 목업을 참고하여 각 슬라이드 디자인
5. 웹사이트 소스 코드에서 필요한 코드 스니펫 추출하여 적용
6. 웹사이트 스크린샷 및 기능 시연 이미지/GIF 준비
7. 애니메이션 및 전환 효과 적용
8. 최종 검토 및 수정

## 필요한 스크린샷 및 시각 자료
프레젠테이션에 필요한 스크린샷 및 시각 자료는 다음과 같습니다:

1. **웹사이트 구조**:
   - 메인 페이지 전체 스크린샷
   - 사이트맵 다이어그램

2. **반응형 디자인**:
   - 데스크톱 버전 스크린샷
   - 모바일 버전 스크린샷
   - 모바일 메뉴 열림/닫힘 상태 비교

3. **네비게이션 시스템**:
   - 스크롤 전/후 헤더 변화 비교
   - 네비게이션 구조 다이어그램

4. **메뉴 브라우징**:
   - 메뉴 카테고리 네비게이션 스크린샷
   - 스크롤에 따른 카테고리 활성화 변화 (가능하면 GIF)

5. **매장 위치 찾기**:
   - 네이버 지도 API 통합 스크린샷
   - 매장 목록 스크린샷

6. **이벤트 및 프로모션**:
   - 이벤트 카드 디자인 스크린샷
   - 메인 페이지와 이벤트 페이지 연동 예시

7. **기술 스택**:
   - 기술 스택 구조 다이어그램
   - 성능 최적화 비교 차트

8. **향후 개발 계획**:
   - 개발 로드맵 타임라인
   - 주요 기능 목업 이미지

## 코드 스니펫 준비
프레젠테이션에 포함할 주요 코드 스니펫은 다음과 같습니다:

1. **모바일 메뉴 토글**:
```javascript
// 모바일 메뉴 토글
if (mobileMenuToggle) {
    mobileMenuToggle.addEventListener('click', function () {
        this.classList.toggle('active');
        mainMenu.classList.toggle('active');
    });
}
```

2. **스크롤 반응형 헤더**:
```javascript
// 헤더 스타일 변경
if (scrollY > 100) {
    header.classList.add('scrolled');
} else {
    header.classList.remove('scrolled');
}
```

3. **스무스 스크롤**:
```javascript
window.scrollTo({
    top: Math.max(0, targetY),
    behavior: 'smooth'
});
```

4. **네이버 지도 API 통합**:
```javascript
var map = new naver.maps.Map('map', {
    center: new naver.maps.LatLng(37.3954, 127.1119),
    zoom: 15
});

var marker = new naver.maps.Marker({
    position: new naver.maps.LatLng(37.3954, 127.1119),
    map: map
});
```

## 발표 팁
- 전체 발표 시간은 20-30분으로 계획
- 각 슬라이드당 2-3분 내외로 설명
- 코드 스니펫을 설명할 때는 핵심 부분을 강조
- 기술적 용어를 사용할 때는 간략한 설명 추가
- 가능하면 실제 웹사이트 라이브 데모 준비
- 질의응답 시간 5-10분 추가 고려

## 대상 청중별 조정 사항
프레젠테이션은 대상 청중에 따라 내용과 강조점을 조정할 수 있습니다:

- **기술 팀**: 코드 구현 세부사항과 기술적 결정 이유에 중점
- **경영진**: 사용자 경험 향상과 비즈니스 가치에 중점
- **클라이언트**: 기능적 특징과 사용자 혜택에 중점
- **디자이너**: UI/UX 요소와 시각적 피드백 메커니즘에 중점

## 추가 참고사항
- 프레젠테이션은 웹사이트의 현재 기능뿐만 아니라 향후 개발 가능성도 포함
- 실제 사용자 시나리오를 통해 기능의 유용성 강조
- 기술적 구현과 사용자 경험 사이의 연결성 설명
- 경쟁사 웹사이트와의 차별점 강조 (해당되는 경우)

이 가이드라인을 따라 제작된 프레젠테이션은 찰리스버거 웹사이트의 기능적 측면을 효과적으로 전달할 수 있을 것입니다.
